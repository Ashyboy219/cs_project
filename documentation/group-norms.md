## Establish Group Communication

**If someone is absent:**  
We message in our iMessage group chat the same day. The person in class posts what we did in the period and what still needs doing. The absent partner catches up before the next class by reading GitHub commits and the record of tasks.

**How we share files electronically:**  
- **GitHub** — main place for code and documentation (`documentation/` and `product/` folders)  
- **iMessage** — quick questions and reminders  
- **OneNote** (class collaboration) — problem statement and objectives for teacher checkpoints  
- **Pull requests** — partner reviews code before it goes to `main`

We reply to messages within **24 hours** on school days.

## Establish Team Norms — Group Contract

Principles we agree to follow. **Everyone in the group agrees.** If a norm is broken, the consequence is listed below.

| # | Norm | Consequence if broken |
|---|------|------------------------|
| 1 | **Show up prepared** — know your task for the day before class starts | Partner covers critical work that day; person who missed prep finishes it that night and updates the record of tasks |
| 2 | **Commit work regularly** — push meaningful progress at least every project class | No merge until missing work is pushed; we tell the teacher if it happens twice in a row |
| 3 | **Review partner's code** — comment on or approve pull requests before merging to `main` | Direct commit to `main` gets reverted; we pair in class until reviews are caught up |
| 4 | **Keep documentation updated** — when code changes, update spec/testing/tasks within 2 days | Partner writes the doc section; both lose demo credit until docs match the product |
| 5 | **Communicate blockers early** — if stuck for more than 15 minutes, ask partner or teacher | We stop solo spiraling; switch pairs or split the problem in class |
